# RUNBOOK — Bomnae Helper v3 실행 절차

> ⚠️ CC 프롬프트는 **반드시 한 세션에 하나씩** 붙인다. 여러 개를 한 번에 붙이면 에이전트가 범위를 섞는다(사고 이력).

## 0. 사전 배치 (Claude Code 실행 전, 손으로)

1. 새 레포 루트에 이 팩의 파일을 배치:
   - 루트: `DESIGN.md` `IA.md` `ROUTES.md` `COMPONENTS.md` `PATTERNS.md` `AGENTS.md` `PROGRESS.md` `tokens.js`
   - 기존 고정 `CLAUDE.md`를 루트에 복사 (이 팩에는 없다 — 훼손 금지 원칙)
2. AGENTS.md는 v3로 교체본이다 — 이식 문서 §10("문서 팩 v3 신규 작성, AGENTS 포함") 근거. 고정본의 구조 원칙은 전부 계승했고 담당 범위만 Bomnae Helper로 재조준했다. 구 고정본을 유지하고 싶으면 구본의 PHASE 분배·CHECKLIST가 링크인바이오 프로젝트(LinkCard·EventsSection·680px 컨테이너) 기준이라 이 프로젝트를 검증할 수 없다는 점만 인지하고 결정해라.

## 1. 개발 순서 (한 프롬프트 = 한 세션)

| 순서 | 프롬프트 | 선행 조건 |
|---|---|---|
| 1 | CC_PROMPT_0_SETUP | 문서 배치 완료 |
| 2 | CC_PROMPT_1_AGENT1_FOUNDATION | [A0] 커밋 |
| 3 | CC_PROMPT_2_AGENT2_GATE **∥** CC_PROMPT_3_AGENT3_LOOP | [A1] 커밋. 두 개는 병렬 가능(다른 세션·다른 워크트리 권장), 서로의 파일을 건드리지 않음 |
| 4 | CC_PROMPT_4_AGENT_REVIEW | [A2]·[A3] 커밋 전부 |
| 5 | CC_PROMPT_5_AGENT_SERVER | [AR] 커밋 + .env 준비(Neon, Google OAuth) |

- 세션이 85%에서 멈추면: 새 세션에 같은 프롬프트를 다시 붙인다 — 헤더의 PROGRESS.md 읽기가 이어하기를 보장한다.
- 배포: 프론트 Vercel / 서버 Render / DB Neon. PHASE 3 이후.

## 2. 현장 과제 (3~4일차 탐방 = 파일럿) — 개발과 병행

**GPS 좌표 수집** — stops.js의 전 좌표가 PLACEHOLDER다. 각 정류장 도착 시 지도앱 핀 → 좌표 기록 → 탐방 후 stops.js 교체(값만 바꾸면 됨).

**촬영 샷리스트** (피칭 실운행 증빙 + 웹 에셋 겸용):
- 차내: 스토리 클립 시청 장면(외국인 팀원 포함 앵글) — Pilot 페이지 메인
- 도착: 웨이팅 없이 나와 있는 감자빵(선주문 대행의 증거 샷) — Pilot + Home
- 히어로: 텍스트 없는 실사(라인 뷰 — 소양강댐 전경, 감자밭 외관 등 가로 와이드)
- 정류장별 대표 사진 1장 (stops 데이터용, 가로 4:3)
- 사장님 스토리 클립: 감자밭 우선(일 안 되면 자막 카드로 대체 가능하게 인터뷰 오디오라도)
- **오늘 밤 WhatsApp 공지**: 차내 촬영 담당 / 도착 샷 담당 역할 분담 (이식 문서 말미 점검사항)

**5일차 확정 대상**: 라인 가격·회차(현재 DRAFT: Adult ₩18,000 / Child ₩12,000 / 주말 +₩3,000, 회차 10:00·13:00·16:00), Hands-Free 요금(₩25,000/개), 영문 서비스명 최종.

## 3. 피칭 연결점 (이 레포가 증명하는 것)

- 대전 빵티칸 순례버스는 여행상품(특별약관·통신판매 등록)으로 합법 판매 중 — 우리 법 리스크 해소 경로의 살아있는 선례 (IA §5).
- 그 상품의 실사용자 리뷰가 45인 대형버스의 구조적 실패(대기줄 역증가·품절)를 지적 — 12인 밴 + 선주문 대행의 차별화 근거 (IA §5).
- 5팀 중 실제 배포된 예약 플랫폼 + 실운행 영상을 가진 팀은 우리뿐 — /pilot 페이지가 그 증거 화면이다.
